一、Broker 网络模型架构

1.1 整体处理流程

  • Netty eventLoopGroupBoss 作为 acceptor 负责接收客户端的连接请求
  • Netty eventLoopGroupSelector 负责 NIO 的读写操作
  • NettyServerHandler 读取 IO 数据，并对消息头进行解析
  • Dispatch 过程根据注册的消息 code 和 processsor 把不同的事件分发给不同的线程进行处理, 此处针对不同的业务类型，分片了特定的处理线程池，不同业务之间能够做到线程隔离。

二、源码分析

1、BrokerController#initialize

Broker启动加载数据并创建线程池

注册各种Processor

2. NettyConnectManageHandler

NettyConnectManageHandler 进行连接管理，监听连接各种状态 ，并发送相应的事件，由ChannelEventListener监听并处理。

3. NettyEventExecutor

接收NettyEvent 表基于事件类型，调用Listener的特定方法处理.
